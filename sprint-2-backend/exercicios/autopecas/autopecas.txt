A empresa AutoPeças convidou você a desenvolver uma nova API para que seja possível realizar a disponibilização dos registros de suas peças e seus determinados fornecedores.
Devemos considerar os critérios mencionados abaixo:
- Tanto administradores quanto funcionários poderão ler, atualizar, cadastrar e deletar peças;
- Tanto administradores quanto funcionários poderão listar todos os fornecedores;
- Somente administradores poderão cadastrar, atualizar e deletar fornecedores;
- Somente administradores poderão cadastrar novos usuários.

Você deverá ser capaz de desenvolver os seguintes tópicos abaixo:

+ Banco de Dados

Script contendo a criação do banco, das tabelas e seus determinados registros.
- 01_autopecas_script.sql
M_AutoPecas | T_AutoPecas
Tabelas contendo os seguintes dados:
- Peças - código da peça (que o administrador/funcionario define), descricao, peso, preço de custo, preço de venda e somente um fornecedor vinculado;
- Fornecedores - cnpj, razão social, nome fantasia, apenas um email de contato e apenas um telefone de contato. Um endereço vinculado;
- Usuários - email, senha e somente uma permissão vinculada.

+ BackEnd
Deverá conter os seguintes endpoints:

GET /api/pecas
GET /api/pecas/{id}
POST /api/pecas
PUT /api/pecas/{id}
DELETE /api/pecas/{id}

GET /api/fornecedores
GET /api/fornecedores/{id}
POST /api/fornecedores
PUT /api/fornecedores
DELETE /api/fornecedores/{id}

Lembrar de incluir o swagger e lembrar de colocar mais um endpoint para gerar o token e outro para cadastrar novos usuários.
Lembre-se de limitar os endpoints para que somente usuários autenticados e autorizados realizem a listagem, cadastro e afins.

Deverá ser entregue em um repositório do github e incluir o link na planilha disponibilizada para que vocês coloquem na coluna correspondente (AutoPeças).

Dados a serem entregues:
- BD
- BackEnd
- Swagger
- Postman
- GitHub

+ Desafios Extras

Disponibilizar um endpoint para que seja mostrado o valor de ganho que a autopeças irá obter calculando o preço de venda sob o preço de custo. Mostrar a porcentagem e um outro campo mostrando a diferença em reais entre um e outro.
GET /api/pecas/precos

Disponibilizar um endpoint para mostrar todas as peças de um determinado fornecedor.
GET /api/fornecedores/{id}/pecas

Disponibilizar um endpoint para listar todos os usuários (mas não mostrar a senha. Somente email.). Somente administradores poderão visualizar.
GET /api/usuarios
